# 楽器

::: warning 楽器ごとの物理モデリングは削除されました
以前のバージョンには、各ノートの演奏可能性をスコア化する詳細な物理演奏モデル（ボウイング方向、手の幅、フレット位置、疲労、ハーモニクスなど）が含まれていました。作曲エンジンはこれらを**使用しません**。楽器の扱いは意図的にシンプルになりました — General MIDI プログラム、演奏可能音域、装飾密度傾向です。このページはエンジンが今日実際に行うことを説明します。
:::

## 楽器が選択するもの

`instrument` の選択は次の3つに影響します。

1. **General MIDI プログラム** — MIDI ファイルに書き込まれる音色。
2. **演奏可能音域** — 候補探索が生成された線を収める音域の窓。ノートはエルゴノミクス的な難易度をモデル化されるのではなく、音域内に折り返されます。
3. **装飾密度** — オプトインの装飾処理が線をどれだけ濃密に装飾するか（`character` と組み合わせて）。

声部数・拍子・構造は `instrument` ではなく `form` から決まります。各形式には既定の楽器がありますが、任意の楽器に差し替えできます。

::: info 楽器と声部
**楽器**は再生音色と音域傾向です。**声部**は生成される音楽上の線です。オルガンのように、一つの楽器が複数の声部を担うこともあります。
:::

## General MIDI プログラム

| 楽器 | 文字列 | GMプログラム | 音色 |
|------|--------|------------|---------|
| オルガン | `"organ"` | 19 | チャーチオルガン |
| チェンバロ | `"harpsichord"` | 6 | チェンバロ |
| ピアノ | `"piano"` | 0 | アコースティックグランドピアノ |
| ヴァイオリン | `"violin"` | 40 | ヴァイオリン |
| チェロ | `"cello"` | 42 | チェロ |
| ギター | `"guitar"` | 24 | アコースティックギター（ナイロン） |

不明な楽器文字列は既定へ暗黙的に代替せず却下されます（例外を投げます）。

## 形式ごとの既定楽器

| 形式 | 既定楽器 |
|------|--------------|
| `fugue`、`prelude_and_fugue`、`trio_sonata`、`chorale_prelude`、`toccata_and_fugue`、`passacaglia`、`fantasia_and_fugue` | オルガン |
| `cello_prelude` | チェロ |
| `chaconne` | ヴァイオリン |
| `goldberg_variations` | チェンバロ |

## 装飾密度

決定論的な装飾処理がトリル、モルデント、ナッハシュラークを追加します。どれだけ濃密に装飾するかは `character` と `instrument` の両方に依存し、一部の線は決して装飾されません。

::: info トリル、モルデント、ナッハシュラーク
これらは構造上の音の周囲に置かれる短いバロック装飾です。主な作曲結果を検証した後に追加されるため、`"material"` や `"compose"` ではなく `source: "ornament"` として記録されます。
:::

- グラウンドバスの線（パッサカリア、シャコンヌ、ゴルトベルクのバス）は決して装飾されません。
- 定旋律の線（コラール前奏曲）は決して装飾されません。

ここで追加されたノートは `source: "ornament"` 由来タグを持ちます。

## 表現の出力

装飾に加え、エンジンは形式にふさわしい表現を書き込みます。

- **オルガンレジストレーション** — 形式のエネルギーアークに従うCC#7/#11カーブ。
- **終結のリタルダンド** — 最終カデンツを遅くするテンポイベント。
- **拍子記号** — パッサカリアとシャコンヌは3/4、それ以外は4/4。

::: tip
声部が楽器とトラックにどう割り当てられるかは[声部アーキテクチャ](/ja/docs/voice-architecture)を、装飾と表現の処理がどこに位置するかは[生成パイプライン](/ja/docs/generation-pipeline)を参照してください。
:::
